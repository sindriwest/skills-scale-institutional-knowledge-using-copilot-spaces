# OctoAcme Project Management Docs

## Overview

This README gives an at-a-glance summary of OctoAcme's project management processes and links to all foundational program and project management guides in this repo.

### What is OctoAcme's project management approach?

OctoAcme uses an iterative, data-informed project management lifecycle that emphasizes stakeholder alignment, clear roles, up-to-date documentation, process transparency, and continuous improvement. Key principles include:

- **Customer-first prioritization** – every decision centers on delivering customer value.
- **Iterative delivery** – work is broken into small, testable increments to reduce risk and accelerate feedback.
- **Clear ownership** – each project has a named Project Manager (PM) and Product Lead accountable for outcomes.
- **Data-informed decisions** – impact is measured and used to drive continuous iteration.
- **Psychological safety** – teams are encouraged to share feedback, surface risks, and learn from every release.

The lifecycle follows five phases: **Initiation → Planning → Execution → Release → Close & Retrospective**, with defined roles (Developers, Product Managers, Project Managers, QA, Stakeholders), key artifacts (Project Charter, Risk Register, Retrospective notes), and regular communication cadences (standups, weekly PM/PdM syncs, monthly stakeholder updates).
